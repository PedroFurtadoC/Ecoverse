# Parceria UNAERP

## Por que esta seção existe

O Ecoverse nasceu como trabalho final de uma disciplina, mas tem potencial de virar um repositório vivo de educação ambiental usado por turmas futuras na UNAERP. Para que isso aconteça, precisamos formalizar a parceria — e este documento é o convite.

## O que pedimos à UNAERP

1. **Reconhecer a UNAERP como parceira institucional** do projeto, com o nome aparecendo no `LICENSE` e no `NOTICE.md`.
2. **Hospedagem do domínio oficial** (algo como `ecoverse.unaerp.edu.br` ou domínio próprio comprado pela coordenação) — apontando para o deploy automático na Vercel.
3. **Manutenção colaborativa** — alunos de turmas seguintes podem adicionar minigames, quizzes ODS, melhorar a interface, traduzir para outros idiomas, conforme a disciplina enxergar valor.
4. **Apoio à comunicação** — divulgar o uso do projeto em outras turmas, eventos da universidade, semana acadêmica.

## O que a UNAERP ganha

- **Plataforma própria** de educação ambiental, gamificada, mensurável (via leaderboard de turma e estatísticas de progresso).
- **Conteúdo factual citado** com fontes oficiais (UNEP, IBGE, IPCC, MMA) — utilizável diretamente em aulas.
- **Visibilidade no GitHub público** como instituição que apoia desenvolvimento de software acadêmico.
- **Custo zero** — Vercel free tier hospeda o site, Supabase free tier hospeda o leaderboard. Domínio é o único custo opcional.

## O que oferecemos em troca

- **Licença MIT** — a UNAERP pode adaptar, redistribuir, comercializar (se quiser) sem pedir autorização aos autores originais.
- **Documentação completa** — em `docs/`, README e arquivos NOTICE/LICENSE — pra que qualquer mantenedor futuro consiga rodar o projeto sem precisar contatar os autores.
- **Disposição em apresentar** o projeto à coordenação, professoras e direção, e ajudar na transição de manutenção.
